# 3PiRobot Pipeline

This repo teaches a robot to use sonar for obstacle avoidance, modelling how bats might
self-train a navigation policy using imagined (hypothetical) sonar percepts rather than
physical trial-and-error.

**Scientific goal:** demonstrate that, given a means of extracting low-level sonar cues
(IID and distance) from hypothetical local views of the world, a bat-like agent can
derive an obstacle-avoidance policy entirely through "mental simulation".

---

## Step 1 – Data Acquisition  (`SCRIPT_DataAcquisition.py`)

**Purpose:** Collect real sonar echoes with paired ground-truth positions from a physical robot.

**What it does:**
- Drives a real robot through physical arenas using a hand-crafted reactive controller.
- Each step: fires sonar ping → reads corrected IID and distance → decides action.
- Control logic: rotate away from the louder side (sign of IID determines L/R), magnitude
  decreases with distance; random 180° turns prevent getting stuck; step size shrinks near walls.
- Position is tracked externally by `LorexTracker` (camera-based, returns x, y, yaw_deg in mm/deg).
- Saves per-step records: full sonar package, tracker position, executed motion.
- Output: `Data/sessionXXX/` folder (multiple sessions: sessionB01–B05 used throughout).

**Key parameters:** `max_steps`, `turn_probability`, `turn_distance`, `translation_magnitude`, `rotation_magnitude`.

**Dependencies:** `Client` (robot comms), `LorexTracker` (position), `DataStorage` (saving), `ControlParameters` (hand-crafted policy).

---

## Step 2 – Echo Processor  (`SCRIPT_TrainEchoProcessor.py`)

**Purpose:** Train a CNN to extract distance and IID from raw binaural sonar echoes.

**What it does:**
- Input: raw binaural sonar waveform, shape (samples, 2) — two microphone channels (L and R).
- Distance model: 1D CNN (3 conv layers: 32/64/128 filters, kernel 5, max-pool) → scalar
  closest-distance in mm. Trained with Huber loss. Optional linear bias calibration on validation set.
- IID computation: once distance is predicted, a short time window around that distance index
  in the echo is selected; IID = 10·log₁₀(energy_L / energy_R) in dB.
  An optional sweep over window placement (pre/post samples, center offset) finds the window
  that maximises Spearman correlation between computed IID and profile-derived left/right asymmetry.
- Ground-truth distance target: minimum of the geometric profile (from arena wall data) within a
  60° aperture, 21 steps. Data is split by arena quadrant (train: 0,1,2; test: 3) for spatial generalisation.
- Outputs saved to `EchoProcessor/`:
  - `echoprocessor_artifacts.pth` — portable inference artifact (model weights + norm stats + calibration + IID window params).
  - `best_model_pytorch.pth`, `training_params.json`, diagnostic plots.

**Key parameters:** `conv_filters`, `kernel_size`, `dropout_rate`, `iid_window_pre_samples`, `iid_window_post_samples`, `enable_window_sweep`.

**Dependencies:** `Library/DataProcessor.py` (data loading), `Library/EchoProcessor.py` (artifact save/load).

---

## Step 3 – Emulator  (`SCRIPT_TrainEmulator.py`)

**Purpose:** Train a "world model" MLP that predicts (distance, IID) from a geometric distance profile,
effectively replacing the physical robot+sonar with a differentiable simulator.

**What it does:**
- Input: distance profile — minimum wall distances across 21 azimuth bins spanning 60°
  (profile parameters are inherited from the EchoProcessor artifact to ensure consistency).
- Optional feature augmentation: appends summary statistics to raw profile bins —
  global min, argmin (normalised), left/right asymmetry, local slope around minimum, inverse-distance
  centre-of-mass.
- Architecture: shared MLP trunk (128→128→64, ReLU, Dropout 0.1) with two separate output heads:
  - `distance_head` → distance in mm
  - `iid_head` → IID in dB
- Labels: generated by running the trained EchoProcessor on real sonar data (teacher/knowledge-distillation).
  This allows the emulator to learn the mapping profile → (distance, IID) without needing raw waveforms at runtime.
- Loss: weighted multi-task Huber (distance weight 1.0, IID weight 1.5); IID loss uses per-sample
  reweighting to upweight positive-IID, near-zero, and tail samples.
- Optional per-output linear calibration on the validation set.
- Outputs saved to `Emulator/`:
  - `best_model_pytorch.pth`, `training_params.json`, scatter/training plots.

**Key parameters:** `hidden_sizes`, `head_hidden_size`, `distance_loss_weight`, `iid_loss_weight`, `use_feature_augmentation`, `use_iid_sample_weighting`.

**Dependencies:** `Library/EchoProcessor.py` (teacher inference), `Library/Emulator.py` (artifact save/load), `Library/DataProcessor.py`.

## Step 4 – Policy Training via GA  (`SCRIPT_TrainPolicy_GA_HistoryNN.py`)

**Purpose:** Evolve a small neural-network policy for obstacle avoidance using a Genetic Algorithm,
with the emulator replacing the physical robot.

**What it does:**

### Policy (HistoryNNPolicy)
- A small MLP (default 16→8→2) that maps a rolling history of the last N=5 steps to a steering
  decision.
- Input features per history step (6 values): IID (norm), distance (norm), previous rotate1 (norm),
  previous rotate2 (norm), previous executed drive (norm), previous blocked flag.
- Outputs: rotate2 angle (rotate1 is fixed to 0°).
- Action sequence per step: rotate1 (0°) → sonar query → rotate2 → drive 100mm → next sonar query.

### Genetic Algorithm
- Genome: flattened MLP weights (float32 vector).
- Selection: tournament (size 5), elitism (top 8 kept unchanged).
- Variation: BLX-α crossover (rate 0.85), Gaussian mutation (rate 0.2, σ=0.2).
- Fitness components: step reward (survival), drive reward (distance covered), turning cost
  (magnitude + activity deadband + direction switches), proximity shaping (penalty for being
  near walls below warning/collision thresholds).
- Multi-environment training: each genome is evaluated across multiple arena sessions
  (train: sessionB02–B05, validate: sessionB01), started from empirical robot positions.
- Parallel evaluation: episodes evaluated in parallel via `ProcessPoolExecutor`.

### Simulation
- `EnvironmentSimulator` (in `Library/EnvironmentSimulator.py`):
  - `ArenaLayout`: loads wall point cloud from a session, rotates/translates it into the robot
    frame, bins wall points by azimuth to produce a distance profile.
  - `EnvironmentSimulator`: wraps `ArenaLayout` + trained `Emulator` to provide a `get_sonar_measurement(x, y, orientation)` interface and collision-aware drive simulation.
- Collision detection: marches along the drive segment in small steps; stops if any wall point
  is within `robot_radius_mm` (85mm) or if the arena boundary margin is violated.

**Key parameters:** `population_size`, `generations`, `elitism_count`, `history_len`, `hidden_sizes`, `fixed_drive_mm`, `step_reward`, `w_rotate2_cost`, `warning_distance_mm`, `collision_distance_mm`.

**Output:** `policy_training_results_history_nn/` — per-session best-genome files, training curves, trajectory plots.

**Dependencies:** `Library/EnvironmentSimulator.py` → `Library/Emulator.py` → `Library/EchoProcessor.py`.

## Data flow summary

```
Real robot + arena
        │
        ▼ (sonar waveforms + positions)
SCRIPT_DataAcquisition  →  Data/sessionBXX/
        │
        ▼ (binaural echoes + profile ground truth)
SCRIPT_TrainEchoProcessor  →  EchoProcessor/echoprocessor_artifacts.pth
        │                      (CNN: sonar waveform → distance + IID window params)
        ▼ (profile → teacher labels via EchoProcessor)
SCRIPT_TrainEmulator  →  Emulator/best_model_pytorch.pth
        │                 (MLP: profile → distance + IID)
        ▼ (emulator replaces real robot)
SCRIPT_TrainPolicy_GA_HistoryNN  →  policy_training_results_history_nn/
                                     (GA-evolved MLP policy)
```
